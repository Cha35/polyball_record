S-01 기록 작성 진입
직관/집관 기록을 시작하는 두 가지 경로를 정의한다.
화면 개요
기록 시작 진입점
유저가 기록을 시작하는 방법은 두 가지다. 경기 종료 후 앱이 자동으로 알림을 발송하는 푸시 기반 진입과, 유저가 능동적으로 기록탭에서 시작하는 수동 진입이다. 두 방식은 동일한 기록 작성 화면으로 연결되며, 진입 경로에 따라 사전 입력 항목의 자동 완성 여부가 달라진다.
진입 방식 범위 TBD
진입 경로 A — 푸시 알림 기반 (자동 트리거)

경기가 종료되는 시점에 폴리볼 서버가 해당 경기와 연관된 유저에게 푸시 알림을 발송한다. 알림 대상은 해당 경기에 승부 예측 참여 이력이 있는 유저를 기본으로 하며, 향후 GPS 인증 유저(고도화 버전)로도 확장 가능하다.

01
경기 종료 → 서버가 해당 경기 예측 참여 유저에게 푸시 발송
푸시 문구 예시: "오늘 LG vs 두산 경기가 끝났어요. 직관/집관 기록을 남겨보세요 ⚾"
02
유저가 푸시 탭 → 앱 실행 후 기록 작성 화면으로 딥링크 이동
앱이 비활성 상태인 경우 앱 실행 후 해당 화면으로 직접 진입
03
기록 작성 화면 진입 시 해당 경기 정보(날짜·팀·스코어) 자동 완성 상태로 노출
04
직관 여부 선택 UI 노출 — "오늘 직접 가셨나요, 집에서 보셨나요?"
직관/집관에 따라 이후 화면이 분기됨 (S-02, S-03 참고)
⚠️ 푸시 발송 시점(경기 종료 후 즉시 vs 일정 시간 딜레이)은 별도 알림 기획 문서에서 정의한다. 유저가 푸시 알림을 무시하거나 닫은 경우 기록은 생성되지 않으며, 이후 수동 진입으로 소급 가능하다.
진입 경로 B — 수동 진입 (능동 트리거)

유저가 직접 기록 탭 또는 내 지갑에서 "기록 남기기" 버튼을 탭하여 시작하는 방식이다. 푸시를 놓쳤거나 과거 경기를 기록하고 싶은 경우에도 사용된다.

01
기록 탭 또는 내 지갑 화면에서 "+ 기록 남기기" 버튼 탭
02
날짜 선택 → 해당 날짜에 열린 KBO 경기 목록 자동 노출
당일 경기가 없는 날짜 선택 시 "해당 날짜에 경기가 없어요" 안내 문구 표시
03
경기 선택 → 직관/집관 여부 선택 → 기록 작성 화면 진입
선택한 경기의 날짜·팀·스코어가 자동 완성된 상태로 노출
📌 두 진입 경로 모두 동일한 기록 작성 화면(S-02, S-03)으로 연결된다. 차이는 자동 완성 데이터의 유무가 아니라, 직관/집관 선택 UI의 노출 시점과 형태에 있다.

S-02 직관 기록 작성 화면
직관(현장 관람)을 선택한 유저가 기록을 작성하는 화면.
화면 개요
직관 기록 작성
경기 기본 정보는 자동 완성된 상태로 제공되며, 유저는 선택 항목만 추가 입력하면 된다. 아무것도 추가하지 않아도 기록이 완성된 것처럼 느껴져야 하며, 추가 입력은 기록을 풍성하게 만드는 선택지로 제시된다.
S-01에서 진입 S-04로 이동
자동 완성 영역

유저 입력 없이 시스템이 자동으로 채우는 항목이다. 화면 상단에 카드 형태로 노출되며 수정이 불가하다. 단, KBO 데이터 연동 방식에 따라 자동 완성 범위가 달라질 수 있어 플랜 B 처리가 필요하다.

항목데이터 출처표시 방식비고
경기 날짜진입 시점 또는 유저 선택YYYY.MM.DD (요일)
구장명KBO 경기 데이터구장 전체 명칭
홈팀 / 원정팀KBO 경기 데이터팀명 + 홈🏠 아이콘
최종 스코어KBO 경기 결과 데이터홈팀 N : N 원정팀경기 종료 후 자동 반영
승패 결과스코어 기반 자동 계산승 / 패 / 무 텍스트
예측 참여 결과승부 예측 이력✅ 예측 성공 / ❌ 예측 실패예측 참여 이력 없으면 미표시
⚠️ KBO 데이터 연동 불가 시(플랜 B): 날짜·팀은 유저 선택으로 자동 완성, 스코어는 유저가 직접 입력. 연동 확보 후 자동화로 전환.
선택 입력 영역

유저가 원할 때만 추가하는 항목이다. 입력하지 않아도 기록 저장이 가능하며, 각 항목은 독립적으로 추가/제거할 수 있다. 항목이 비어있을 때 화면상 공백이나 미완성 느낌을 주지 않도록 설계해야 한다.

항목입력 방식상세
사진갤러리 선택 또는 카메라 촬영업로드 가능 장수는 정책 별도 정의. 장수 초과 시 유료 포토패스 유도 안내.

S-03 집관 기록 작성 화면
집관(집에서 시청)을 선택한 유저가 기록을 작성하는 화면.
화면 개요
집관 기록 작성
직관 기록 작성 화면(S-02)과 구조는 동일하나, 사진 항목의 우선순위가 낮아지고 선택 입력 항목이 보다 가볍게 구성된다. 집관의 본질은 "이 경기를 진심으로 챙겨봤다"는 팬심의 기록이므로, 부담 없이 빠르게 완성할 수 있는 플로우가 핵심이다.
S-01에서 진입 S-04로 이동
직관 기록 작성 화면(S-02)과의 차이점
항목직관 (S-02)집관 (S-03)
자동 완성 카드날짜·구장·팀·스코어·예측결과날짜·팀·스코어·예측결과 (구장 미표시)
카드 배경 텍스처구장 분위기 배경홈/실내 분위기 배경 (구분 필요)
사진 항목노출 우선순위 높음노출 우선순위 낮음 (하단 배치 또는 생략 고려)
추가 입력 항목사진·소감·동행인·날씨·기분소감·기분 (날씨·동행인 선택적 노출)
기록 카드 표기"직관" 레이블"집관" 레이블
📌 집관 화면은 직관 화면을 기반으로 구성하되, 불필요한 항목을 걷어내어 더 빠르게 완성되는 경험을 제공한다. 두 화면을 하나의 화면에서 분기 처리할지, 별도 화면으로 개발할지는 개발팀과 협의 필요.
집관 인증 처리 방식

GPS 인증 없이 집관 기록을 생성하는 MVP에서는 별도의 인증 절차 없이 유저의 자기 선언으로 처리한다. 승부 예측 참여 이력이 있는 경우 이를 간접 인증 근거로 활용하며, 카드에 표기한다.

조건처리
예측 참여 이력 있음카드에 "✅ 예측 참여" 자동 표기. 간접 인증 근거로 활용.
예측 참여 이력 없음별도 표기 없이 일반 집관 기록으로 처리. 인증 없음 상태.
보상 연동 시v2 시점에 실시간 퀴즈 인증 등 신뢰도 장치 별도 설계. MVP에서는 보상 미연동.

S-04 기록 카드 완성 화면
기록 저장 완료 직후 노출되는 화면. 생성된 카드 확인 및 공유 액션을 제공한다.
화면 개요
기록 카드 완성
기록 저장 직후 유저에게 "카드가 만들어졌다"는 감각을 주는 화면이다. 단순한 완료 메시지가 아니라, 완성된 카드를 실제로 보여주며 소유감을 형성한다. 이 화면에서 SNS 공유 또는 내 지갑 이동의 두 가지 액션으로 자연스럽게 연결된다.

📌 URL 예시: polyball.com/card/00847 · 퍼블릭 페이지에서 카드 정보 + 소유자 확인 가능. 개인정보 노출 범위는 보안 정책에서 별도 정의.

S-07 직관 승률 통계 화면
누적 기록을 바탕으로 개인화된 직관/집관 승률 데이터를 보여주는 화면.
화면 개요
직관 승률 통계
기록이 쌓일수록 의미 있는 개인 통계가 형성된다. 기록이 적은 초반에는 통계 화면을 바로 노출하지 않고, 일정 수(5개) 이상 쌓였을 때 공개되는 방식으로 설계한다. 오픈 전 상태의 프로그레스 바 자체가 기록 추가를 유도하는 리텐션 장치가 된다.
내 지갑 또는 기록 탭에서 진입 잠금 / 오픈 두 가지 상태
상태별 화면 정의
잠금 상태
기록 5개 미만일 때 노출. 통계 수치 대신 프로그레스 바 + "기록 N개 더 쌓으면 내 승률이 열려요" 문구 표시. 통계 영역은 블러 또는 잠금 아이콘으로 처리하여 오픈 후 어떤 정보가 제공되는지 미리 보여준다.
오픈 상태
기록 5개 이상 달성 시 통계 탭 오픈. 최초 오픈 시 "승률이 열렸어요! 🎉" 알림 또는 축하 연출 노출. 이후 진입 시 통계 바로 표시.
오픈 후 제공 통계 항목
항목설명표시 방식
직관 승률직관 기록 중 홈팀(응원팀) 승리 비율퍼센트 (%) 숫자 강조
집관 승률집관 기록 중 홈팀 승리 비율퍼센트 (%) 숫자 강조

가장 많이 간 구장직관 기록 기준 최다 방문 구장구장명 텍스트 + 방문 횟수
