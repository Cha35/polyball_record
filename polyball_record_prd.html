01개요 및 배경

폴리볼은 승부 예측 콘텐츠를 통해 응모권을 획득하고, 이를 통해 직관 티켓을 받을 수 있는 서비스다. 현재 유저는 예측 참여 → 당첨 → 직관으로 이어지는 여정을 경험하지만, 이 경험은 단발성 이벤트에 그치며 이후 유저가 폴리볼로 돌아올 지속적인 이유가 부족하다.

동시에 KBO 직관 기록 시장(야구보구, 직관일기, 나의 야구 일지 등)에서는 서버 불안정으로 인한 데이터 유실이 유저 이탈의 핵심 원인으로 확인된다. 기업 수준의 안정적인 인프라와 승부 예측과 연결된 통합 기록 경험에 대한 니즈가 존재한다.

이에 따라 직관/집관 기록 기능을 신규 개발하여 유저의 팬 경험 전체를 폴리볼 안에서 완결시키고, 이를 내 지갑 리뉴얼과 연결하여 모든 유저가 지속적으로 방문하는 공간으로 전환한다.

📌 기존 응모/예측 기능은 현행 구조 유지. 직관 기록은 신규 기능으로 추가되며, 내 지갑은 기존 당첨 티켓을 유지한 채 확장된다.
02목표
Goals
  • ✅직관/집관 기록이 쌓일수록 유저가 폴리볼을 떠나기 어려운 구조를 만든다 (기록 = 폴리볼에만 존재하는 자산)
  • ✅예측 참여 → 직관/집관 → 기록 → 통계로 이어지는 팬 라이프사이클을 폴리볼 안에서 완결시킨다
  • ✅당첨자에게만 의미 있던 '내 지갑'을 모든 유저가 방문하는 컬렉션 공간으로 전환한다
  • ✅1인 개발 앱 대비 안정적인 데이터 보존으로 기존 기록 앱 유저를 흡수한다
  • ✅응모 당첨 직관 기록에 특별한 가치를 부여하여 당첨 경험의 감동을 극대화한다
Non-Goals
  • ❌기존 응모/예측 메인 UX를 변경하지 않는다
  • ❌감성 모드 / 기록 모드 이중 구조는 이번 범위에 포함하지 않는다 (유저 반응 확인 후 v2 검토)
  • ❌KBO 외 타 리그/종목은 이번 범위에 포함하지 않는다
  • ❌NFT 실제 발행, 카드 거래/교환 기능은 장기 로드맵으로 분류한다

04사용자 시나리오
Scenario A
직관 기록을 남긴다 (핵심 플로우)경기 당일 구장 도착 후 폴리볼 앱 실행. GPS 인증 팝업에서 확인 → 날짜/구장/팀 자동 완성 → 경기 종료 후 "기록 완성됨" 푸시 수신 → 스코어 자동 반영 → 선택적으로 사진·소감 추가 → 기록 카드 생성 및 내 지갑 저장.
Scenario B
집관 기록을 남긴다 (예측 연동 플로우)경기 시작 전 승부 예측 참여 → 경기 종료 후 "집관 기록 남길게요?" 푸시 수신 → 확인 탭 → 스코어 + 예측 결과 자동 완성된 기록 카드 생성.
Scenario C
응모 당첨으로 직관을 간다 (특별 카드 플로우)응모 당첨 후 직관 당일 GPS 인증 → "폴리볼 당첨으로 오신 특별한 날!" 화면 노출 → 이벤트명·도전 횟수 자동 표기 → 일반 카드와 시각적으로 구분된 특별 카드 생성.
Scenario D
내 지갑에서 기록을 탐색한다마이 메뉴 → 내 지갑 진입 → 요약 정보(기록 카드 N장·배지 N개·티켓 N장) 확인 → 타임라인 뷰에서 기록 탐색 → 카드 탭 → 상세 확인 후 공유.

06기능 요구사항
6-1 GPS 기반 직관 인증
📌 목적: 유저가 실제로 구장에 있음을 확인하고, 기록을 자동 시작하는 트리거. MVP에서는 부정 기록 방지보다 편의성 우선 설계.
항목내용
구장 좌표 등록KBO 10개 구장 GPS 좌표 사전 등록 (착수 전 완료 필요)
인증 기준구장 반경 500m 이내 위치 시 인증 성공 처리
데이터 매핑인증 성공 시 해당 날짜의 경기 데이터 자동 연결
GPS 불안정신호 불안정 시 재시도 안내 제공
당일 경기 없음해당 구장에 당일 경기가 없는 경우 예외 처리 별도 정의
티켓 스캔 (부가)모바일/종이 티켓 스캔 시 좌석 번호 자동 입력. GPS 인증의 보완 수단, 필수 아님
⚠️ 예외 케이스 (기능정의서에서 상세 정의 필요): 우천 취소 / 더블헤더 / 경기 연장. 우천 취소 처리 방향 → "경기는 취소됐지만, 구장에 간 기록은 남아요" 로 검토.
✅ 플랜 B (KBO 데이터 연동 불가 시): 경기 일정 파싱으로 날짜/팀 자동 완성 유지. 스코어는 유저 직접 입력으로 임시 처리 후 연동 확보 시 자동화 전환.

📌 카드 설계 원칙: 선택 입력이 하나도 없어도 카드가 완성된 것처럼 보여야 한다. 빈 항목이 "비어있다"는 느낌을 주면 안 됨. 직관/집관 카드는 시각적으로 구분 (구장 배경 vs 홈 배경 텍스처). 모든 카드에 고유 발행 번호 부여. 카드별 공유 URL 생성 (예: polyball.com/card/00847).
6-3 집관 기록 — 예측 연동 트리거
항목내용
트리거 조건해당 경기에 승부 예측 참여 이력이 있는 유저에 한해 경기 종료 후 자동 푸시 발송
자동 완성경기 날짜, 팀, 최종 스코어, 예측 결과 (성공/실패)
특별 표기예측 성공 + 집관 승리 시 카드 내 별도 표기
알림 무시알림 무시 시 기록 미생성 (강제 아님). MVP에서 소급 생성 불가
⚠️ 집관 기록은 GPS 인증이 없어 신뢰도 완전 보장 불가. MVP에서는 예측 참여 이력을 간접 인증으로 사용. 보상 연동 시점(v2)에 신뢰도 장치 강화 예정.

v1.5 · v2 · 장기 방향
11고도화 로드맵
v1.5 — 출시 후 1개월 내
📌 개발 착수는 MVP와 병렬 준비. 출시 후 지체 없이 배포. 우선순위 관리 필수.
v1.5 과거 기록 소급 입력
날짜 + 팀 선택 시 해당 경기 결과 자동 매핑. 직관/집관 여부 선택. 소급 입력 카드에 "소급 입력" 표기 (신뢰도 구분). 입력 가능 기간 범위 정의 필요 (전 시즌까지 허용 여부).
v1.5 구장 스탬프 지도
전국 KBO 구장 지도 표시. 직관 기록이 있는 구장에 스탬프 표기. 전체 구장 방문 프로그레스 표시. 입문 유저 온보딩 및 직관 동기부여 효과.
v2 — 출시 후 3개월 내외
v2 감성 모드 / 기록 모드 이중 구조
기록 시작 시 모드 선택 또는 설정에서 기본값 설정. 감성 모드: 날씨·기분·동행인·소감 중심 5개 이하 질문형 UI. 기록 모드: 선발 투수, 인상적인 장면 등 상세 입력. 유저 반응 확인 후 도입 여부 결정.
v2 카드 등급 세분화 + 홀로그램 인터랙션
이벤트 등급 설정 (일반/레어/에픽/레전더리). 등급별 카드 디자인 차별화. 터치 시 홀로그램 효과 애니메이션 (카드 물성 구현, NFT 느낌 강화).
v2 집관 실시간 퀴즈 인증
이닝 종료 등 이벤트 트리거 자동 팝업. "현재 스코어는?" 형태 퀴즈. 정답 시 집관 인증 강도 상향. KBO 실시간 데이터 연동 전제. 보상 연동 시점에 신뢰도 강화 목적으로 도입.
v2 배지 시스템 고도화
MVP 응모 도전 배지 외 다양한 조건 배지 추가. 예시: 첫 직관 / 10회 직관 / 원정 직관 / 연승 동행 / 전 구장 방문 등. 배지도 내 지갑 컬렉션으로 표시.
v2 응모 당첨 타임라인 스토리
응모일 → 당첨 발표일 → 직관 당일 타임라인 시각화. 당첨 발표 푸시 알림 수신 타임스탬프 기록. 카드 상세에서 스토리 보기 가능.
v2 시즌 Wrapped
시즌 종료 후 자동 생성. 직관 기록 없는 유저도 예측 이력으로 Wrapped 생성 가능. 풀스크린 슬라이드 형태, 팀 컬러 개인화. 시즌 마무리 시점에 맞춰 출시.
v2 첫 번째 종목 확장 파일럿
K리그 또는 KBL 중 하나를 파일럿으로 추가. 카드 테이블 카테고리 구조 검증. 1차 확장 검증 후 2차 확장(V리그 등) 진행.
장기 — 월렛 플랫폼화
장기 카드 선물하기
"같이 갔던 직관 기록 카드를 그 친구한테 선물한다"는 소셜 경험. 동일 경기 카드가 두 사람에게 각각 발행되는 구조 (번호는 다름).
장기 파트너십 기반 카드 발행
공연, 전시 등 타 문화 영역과의 파트너십으로 카드 발행 확장. 폴리볼이 스포츠·문화 경험을 담는 통합 플랫폼으로 성장.
장기 NFT 전환 옵션
유저 선택으로 기록 카드를 블록체인 기반 NFT로 전환. 토큰 게이팅: 특정 카드·배지 보유 유저 대상 전용 혜택 제공. 현재는 NFT 느낌의 UX로 접근하고, 실제 발행은 생태계 성숙 후 도입.
